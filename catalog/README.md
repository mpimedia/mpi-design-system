# Component Catalog

The catalog of all UI components in the MPI design system, organized by complexity.

## Categories

| Directory | What it contains | Atomic equivalent | Examples |
|---|---|---|---|
| [`elements/`](elements/) | Smallest building blocks | Atoms | Buttons, inputs, badges, icons, links |
| [`components/`](components/) | Self-contained UI units | Molecules | Cards, modals, navbars, tables, alerts |
| [`patterns/`](patterns/) | Recurring multi-component arrangements | Organisms | Forms, search bars, filters, data entry |
| [`layouts/`](layouts/) | Full page structures | Templates/Pages | Dashboards, detail pages, list views |

## Cross-Reference

When documenting a component, include:

- **Plain language name** — What MPI calls it (primary identifier)
- **Atomic Design term** — For teams familiar with that methodology
- **Bootstrap 5 component** — The closest Bootstrap equivalent, if any

## Component Status

Each spec includes a status badge:

| Status | Meaning |
|---|---|
| **Draft** | Spec is in progress, not yet reviewed |
| **In Review** | Design is being reviewed by stakeholders |
| **Approved** | Design is accepted, ready for implementation |
| **Deprecated** | Superseded by a newer version, avoid using |

## Finding Components

Browse the directories above, or check the [project board](https://github.com/orgs/mpimedia/projects/15) for components currently in progress.
